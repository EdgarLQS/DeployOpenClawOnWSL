## 优化建议

### 模型使用建议

| 场景 | 推荐模型 | 理由 |
|------|----------|------|
| 简单问答 | DeepSeek-V3 | 响应快，成本低 |
| 复杂分析 | DeepSeek-R1 | 推理能力强 |
| 代码相关 | - | - |

### 成本优化建议

1. **合理设置 thinking 级别**
   - `low`: 简单查询、时间汇报
   - `medium`: 一般分析、新闻总结
   - `high`: 复杂问题、深度分析

2. **定时任务频率建议**
   - 高频任务（1-5 分钟）：仅用于测试
   - 中频任务（30 分钟 -1 小时）：状态检查、时间汇报
   - 低频任务（每天 1-2 次）：新闻总结、日报生成

3. **Token 节省技巧**
   - 使用 `contextPruning.mode: "cache-ttl"` 管理上下文
   - 设置合理的 `ttl` 值（建议 30m-1h）
   - 定期清理会话历史
